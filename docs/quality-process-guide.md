# Quality Process Guide

## Development Workflow

1. Create a feature branch from `main`.
2. Implement the required functionality.
3. Write or update automated tests.
4. Run all tests locally.
5. Create a Pull Request.
6. CI pipeline automatically runs tests.
7. Team reviews the code.
8. Merge only if all checks pass.

## Definition of Done

A task is considered complete only when:

- Code is implemented.
- Automated tests pass.
- No critical defects remain.
- Documentation is updated.
- Code review is completed.

## Quality Principles

- Test critical business flows first.
- Prevent defects early.
- Automate repetitive regression scenarios.
- Keep the CI pipeline fast and reliable.
