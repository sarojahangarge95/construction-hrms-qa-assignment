# Quality Reflection

## 1. Personal Testing Habit

In my personal projects, I have not consistently written automated tests. Most of my focus has been on learning software testing concepts and automation tools. This assignment helped me understand the importance of testing critical business flows and building automated safety nets.

## 2. Bug Found Late

During a project, I found that incorrect input validation allowed invalid data to be saved. The issue was discovered late because negative testing had not been performed earlier. I learned that testing should start early and include edge cases.

## 3. One Thing This Team Needs Most

The team needs a mandatory CI pipeline. Every code change should run automated tests before merging. This prevents broken code from reaching production.

## 4. Working with the Senior Developer

In my first week, I would demonstrate value by automating a critical payroll flow and showing how automated tests can quickly detect regressions. This helps developers move faster and reduces manual effort.

## 5. Personal System Example

I use daily task lists to organize my learning and interview preparation. The system works because it is simple and easy to follow. Similarly, quality processes should be lightweight and practical so that the entire team follows them consistently.
